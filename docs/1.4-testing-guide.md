# BMA 1.4 — RC Testing Guide

A hands-on walkthrough of everything that changed since **v1.3.0**, so you can drive
it yourself and decide **ship-as-is** vs **fixes-needed** before we regen the manuals
and push. Treat it as a release-candidate checklist — mark each item ✅ / ⚠️ (minor) /
❌ (blocker) as you go; the bottom has a verdict box.

> **Why this comes before the manuals:** manual screenshots must capture the *final*
> UI. If your pass turns up fixes, regenerating manuals first would mean redoing them.
> So: this guide → your RC pass → fixes → manuals → push.

## Setup

```bash
node build.js          # assemble src/ → index.html
# open index.html directly in a browser (Edge/Chrome; OPFS + FSAA need a Chromium engine)
```

**Test data already generated for you** (gitignored, under `experiments/`):
- `experiments/test-project-folder/` — a **samples-as-points** project folder (model + a comparison point set + a cross-tab).
- `experiments/test-project-folder-dh/` — a **drillhole** project folder (model + a trio that re-composites + emits collar / 5 m composite / raw intervals).
- `experiments/BMA-showcase-1.4.bma.zip` — a packed showcase project (for the pack-import path).

Regenerate any of them with `node experiments/make-test-folder.js` /
`make-test-folder-dh.js` / `make-showcase-pack.js`.

> Tip: each test starts clean if you **close the file** (File ▸ Close file, or the ✕)
> to return to the landing. To wipe all local state, clear the site's IndexedDB +
> localStorage in DevTools.

## 1 · Project manager (the new landing) — C14

The landing is now a **project manager**, not a recent-files list.

- [ ] **It renders.** Drop the example model (or `⤓ Download example dataset` then drop it), analyze, then **Close file**. The landing shows a **Projects** list with the project you just worked on (Storage = `local`).
- [ ] **Search / sort / tags / notes.** Use the search box; change the **Sort** dropdown (Last opened / saved / created / title / model). Click **✎** on a project → add a tag and a note; reopen the editor → they persisted. A tag chip appears + a tag-filter button at the top.
- [ ] **Responsive.** Narrow the window below ~700px → the table collapses to stacked cards; widen → back to columns.
- [ ] **Import a pack — pick a backend.** Click **⬇ Import pack** → choose `BMA-showcase-1.4.bma.zip` → a small dialog offers **Unpack to a folder / Browser storage (OPFS) / Embedded (IDB)**. Pick **Embedded** → it appears in the list (badge `embedded`) **without opening**. Click **Open** → it opens. (Try OPFS too — badge `browser storage`.)
- [ ] **Re-import = a NEW project (never overwrites).** Import the same pack again → a second entry appears, titled `… (copy)`. The first one (and any edits to it) is untouched.
- [ ] **Open / Backup / Delete** per row. **⤓** backs up as a `.bma.zip`. **✕** removes (a stored OPFS/IDB copy is deleted; a folder is only forgotten).
- [ ] **File menu.** With a project open: **File ▸ Open recent** lists projects (with storage icons) + an **All projects…** footer → returns to the landing manager. Picking a different recent **switches** projects cleanly.

## 2 · Folder-backed projects — C11

- [ ] **Mount a folder → it opens itself.** On the landing, **📁 Open project folder** → pick `experiments/test-project-folder` → the whole project opens from disk (model + the samples comparison + a cross-tab), **no drop**. Header shows a 📁 folder pill.
- [ ] **Edits persist to disk.** Change something (rename a view, edit a filter) → it autosaves into the folder's `*.bma.json`. (Re-mount later → the change is there.)
- [ ] **Recents.** Close the file → the folder is in the **Projects** list with a 📁 **folder** badge (one entry only — not also `model.csv`).
- [ ] **Refresh behaves.** Reload the page → you land on the **landing**, not auto-reopened. The folder is in the list for one-click reopen.
- [ ] **Drillhole folder.** Mount `experiments/test-project-folder-dh` → model + a **2 m composited** drillhole set + **three emitted derivations** (collar / 5 m composite / raw intervals) all re-derive from disk. (See §6 for what to check on the emits.)

## 3 · Catalog tree — hierarchy + per-dataset affordances

- [ ] **Derived datasets NEST.** In the drillhole folder, the Data rail shows the drillhole **set** with its emitted datasets (collar / composites / intervals) **indented underneath it** on a connector rail — not as flat siblings.
- [ ] **"not analyzed" is actionable.** A comparison/derived dataset that isn't analyzed shows an **Analyze ▸** button in its tree header (and an **Analyze** item in its right-click menu). Click it → it analyzes, unlocking stats / pairing / views.
- [ ] **Right-click anywhere in the Data panel** gives a sensible menu (dataset actions on a dataset node, variable actions on a variable row, Add-dataset on empty space) — never the raw browser menu.

## 4 · Views (the ontology rethink) — C10

A **View** = an analysis you deliberately keep over a dataset. (We retired the word
"surface".)

- [ ] **The Dataset picker is discoverable.** Open Statistics with only the model analyzed → no picker (it's implicitly the model). Now load a 2nd dataset but **don't** analyze it → the picker appears **disabled** with a hint *"Analyze another dataset to target it here."* Analyze it → the picker goes live with both datasets.
- [ ] **Views group per dataset.** Each analyzed dataset has a **Views** group in the tree. A fresh one is empty with a learnable hint + **+ New view**. Default panels (a pristine Statistics tab) are **not** listed — only deliberate ones.
- [ ] **Create / Duplicate / Delete / Rename.** **+ New view** → pick a kind → it's created targeting that dataset and appears in the group. Rename it (✎ or right-click ▸ Rename) → **the tab title changes to match**. Duplicate (⎘) → an independent copy. Delete (✕) → a clone is destroyed; a renamed singleton "stops being kept".
- [ ] **Right-click a view row** → Focus / Rename… / Duplicate / **Target dataset ▸** (retarget submenu) / Delete.
- [ ] **Universal.** Views work on **every** dataset (model + comparisons + emits), not just the model.
- [ ] **Summary stays put.** A dataset's **Summary / grid geometry** lives with the dataset (its panel), never as a "view".

## 5 · Statistics panel layout

- [ ] **Draggable split.** In Statistics, add a CDF curve (click a column name). Drag the **handle** between the table and the plot → the split resizes; the height persists across reloads.
- [ ] **Plot fills its pane.** The CDF/Prob/Q–Q plot fills the lower pane and **doesn't scroll weirdly** (the old width-only sizing is gone). Resize the window → it re-fits.
- [ ] **Wide table.** Scroll the stats table right (many percentile columns) → the **header row** and the **first (Column) column** stay pinned.
- [ ] **Selection restyle.** Metric toggles are **pills** (active = solid accent fill); the variable list has **custom checkboxes**.

## 6 · Derived-dataset lifecycle (the data-fidelity piece)

The headline: a view over a composite must **survive a reload**.

- [ ] **Emit auto-analyzes.** Load the drillhole trio (or mount the dh folder), emit a composite → it **auto-analyzes** (no manual Analyze needed; it's ready to use).
- [ ] **A view over a composite resolves.** Create a Statistics/GT view targeting an emitted composite. **Close the file and reopen the project** (mount the dh folder again) → the composite re-derives, re-analyzes, and **the view still targets it** (it does NOT silently bounce to the model).
- [ ] **Dependents badge.** A derived dataset that feeds views shows an `N ▦` badge in the tree.
- [ ] **Broken warning.** Open a project whose source ISN'T available (e.g. a loose project without the trio): the derived dataset shows a **⚠** badge, and a single dismissible **banner** appears summarizing *"N derived datasets couldn't be recreated — M views affected…"*. (Hard to stage casually — flag if you can't trigger it; it's covered by smoke.)
- [ ] **Materialize.** Right-click a derived dataset ▸ **Materialize (freeze)** → it becomes ◆ materialized (loads its snapshot, no re-derive). **Relink** reverts. (A slow-to-recreate one also shows a one-click **⚡ materialize?** nudge.)

## 7 · A11 drillhole container / emit

- [ ] **Load without compositing.** Drop a trio → you can **emit the collar** as points *without* compositing first.
- [ ] **Multiple composites coexist.** Emit composites at different lengths (e.g. 2 m and 5 m) from one set — both as their own datasets, no reload.
- [ ] **Raw intervals + merge.** Emit raw intervals (desurveyed to points, weighted by LENGTH). The container toolbar (length / re-composite / emit / report / edit) is always visible.

## 8 · Cross-cutting / regressions to spot-check

- [ ] **No landing bleed-through.** With a project open, the landing/manager is fully hidden — nothing overlaps the workspace (this was a bug).
- [ ] **Cross-tab (A19), GT, Swath, Categories, Export, StatsCat** all still work and can target a comparison dataset + clone.
- [ ] **Pack round-trip.** Pack a project (File ▸ Pack), then import the pack → it restores faithfully (datasets, drillholes, views, layout).
- [ ] **Themes / UI scale** (View menu) still apply.

## 9 · Storage backends & FSAA — **HUMAN-ONLY** (the smokes physically can't cover these)

> Why these are on you: headless Playwright can't drive the OS folder/file pickers,
> can't grant/persist a real File System Access **permission**, and a mock handle
> can't be both IDB-cloneable *and* functional. So the automated suite mocks the
> handles — these checks exercise the **real** browser machinery the mocks stand in
> for. Use a Chromium engine (Edge/Chrome). **Each is a potential silent-data-loss
> path, so treat a failure here as a blocker.**

- [ ] **Real folder grant.** Landing ▸ **📁 Open project folder** → the real OS picker opens → pick an **empty** folder → work a project (analyze, add a comparison) → look in the folder on disk: a `*.bma.json` (and any materialized/exported files) are really there.
- [ ] **Edit-in-place, no snapshot loss.** With that folder project open, rename a view / change a filter → wait a beat → confirm the **"Saved ✓"** beat. Open the `*.bma.json` from disk → the edit is in it (the autosave wrote **into** the folder, not just localStorage).
- [ ] **Write-failure is surfaced (R2).** Hard to stage casually, but if you can revoke folder write access mid-session (or fill the disk), a failed save must show **"Save failed — project folder not writable"**, never a lying "Saved ✓".
- [ ] **Permission regrant after a full restart.** Close the folder project, **quit the browser completely**, reopen, landing ▸ open that folder project from the **Projects** list → expect a **one-click re-grant** prompt (not a silent failure, not a re-pick of every file). Grant → it reopens from disk.
- [ ] **OPFS reopens unattended.** Import a pack to **Browser storage (OPFS)** → reload the page → open it from the manager → it reopens with **no permission prompt** (origin-private). Edit → reload → the edit persisted.
- [ ] **IDB embedded, edit-in-place.** Import a pack to **Embedded (IDB)** → open → rename a view → reopen → the rename persisted (per-file blobs were rewritten in place, not lost).
- [ ] **File-handle reopen (no re-pick).** Open a model via a real **file picker** (where offered) → Close file → reopen from the **Projects** list → it loads **without** re-picking the file (the handle was stored). Contrast a **drag-dropped** model: reopening that one *does* ask you to re-pick (no handle to store) — that's expected, not a bug.

## 10 · Review-fix sanity checks (the data-loss cluster — verify by hand)

> These are the multi-agent review's confirmed silent-loss bugs, now fixed + smoke-
> guarded. A 60-second human pass over the worst ones is worth it.

- [ ] **R1 — no replace-model clobber.** Open project **A** from a folder. Without closing, load a **different** model file. Reopen **A** → its saved JSON is intact (the new model did **not** overwrite A's folder file).
- [ ] **R5 — projects, not files.** Drop a model, work it, Close. Drop the **same** model file again → it's a **new, blank** project (not a silent resume), and the manager shows **two** entries for that file — no collision. To resume the first, open it from the manager.
- [ ] **R3 — derived config survives reload.** Put a filter/calcol on an emitted composite, then reopen the project (re-mount its folder) → the composite's filter/calcol come back (not dropped).
- [ ] **Model-less reopen.** Create a **datasets-only** project (no model) via the empty-project path, name it, Close → from the manager **Open** it → the **workspace actually comes back** (this silently showed a blank before the fix).
