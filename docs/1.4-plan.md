## Two tracks

1.4 runs two independent tracks that don't touch the same code, so they proceed in
parallel.

### Track 1 — C12 Derivation lineage (platform keystone)

The write-side twin of C10's DatasetSource. Sequenced like C10 went in: low-risk,
smoke-guarded, no big-bang. Design of record: `docs/derivation-lineage.md`.

- **C12-P0 — name the contract, retrofit inert.** Define
  `Derivation {id, sources[], derive(), stale, mode}`. Wrap what *already is* a
  derivation — the drillhole composite (recipe + `dhCompositeAndLoad`) and aux
  re-analyze — onto it, **bit-identical**. The C10-P0 move. Guard: existing
  drillhole/aux smokes stay green.
- **C12-P1 — staleness propagation + one Refresh surface.** Topological
  "source changed → downstream stale" walk + a single consistent **stale badge +
  Refresh** affordance that subsumes today's three hand-rolled mechanisms
  (`dhStale`, `markAuxStale`, model re-analyze). The genuinely new capability;
  no-silent-stale (A9) applied to derived data.

**P0 + P1 is the 1.4 ceiling for this track.** Held to 1.5:

- **C12-P2** — emit / materialize / re-export built *as* Derivations (unblocks A11
  emit-as-dataset + A18 bound output).
- **C12-P3** — the `air` expression IR + pushdown, paired with **B2** (parquet).
  Large; chains to the format keystone.

### Track 2 — A19 categorical crosstab (ships now, zero platform deps)

The shippable slice: two categorical columns in **one** dataset (CLASS_2024 ×
CLASS_2025, LITO × DOMAIN, RESOURCE × RESERVE) — same rows, trivial pairing. Four
views of one cross-tabulation surface:

1. cross **table** — counts + row% / col% / total% margins
2. **confusion-matrix** heatmap — diagonal = agreement when it's the same
   classification twice; + agreement %, Cohen's κ
3. **Sankey** — the matrix as flows (year-over-year / reserve↔resource
   transitions = the reconciliation use case)
4. grouped / stacked **bars**

**BMA twist: tonnage-weighting** — a transition matrix by tonnes, not block count,
is the geologically meaningful version (reuses the existing weight machinery).

Built as a **cloneable analysis surface** on the C10 facet `categorical`, so it
plugs into the shared picker / clone / persist plumbing. Worker: an additive 2-col
joint-count (group-by pairs) pass.

The cross-**dataset** A19 (model class vs a re-estimate) stays deferred — it needs
C13 row-pairing (1.5).
