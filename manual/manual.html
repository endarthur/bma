1O que é o BMA

O BMA (Block Model Atelier) é um inspetor de modelos de blocos que roda inteiramente no seu navegador. Você arrasta um CSV exportado do Isatis, Vulcan, Surpac, Leapfrog, Datamine ou qualquer outro pacote de mineração e recebe, em segundos: estatísticas descritivas, detecção de geometria do grid, distribuição de categorias, curvas teor–tonelagem, perfis direcionais (swath) e comparação contra as amostras que deram origem ao modelo — ou contra qualquer número de outros conjuntos de dados.

Três propriedades definem a ferramenta:

  • Nada sai do seu computador. Não existe servidor: o arquivo é lido localmente, via streaming. Pode usar com dados confidenciais.
  • Arquivos enormes funcionam. O BMA nunca carrega o arquivo inteiro na memória — ele processa linha a linha. Modelos de vários gigabytes rodam em um notebook comum. A velocidade é limitada por ler o arquivo uma vez, não pela RAM: espere alguns milhões de linhas em segundos e arquivos de vários gigabytes em (poucos) minutos numa máquina típica — arquivos mais largos (muitas colunas) e colunas calculadas somam a isso. Uma barra de progresso acompanha tudo e você pode cancelar a qualquer momento.
  • Uma passada só. Tipos, estatísticas, geometria e colunas calculadas são computados em uma única leitura do arquivo, em uma thread separada (Web Worker), então a interface nunca trava.

O BMA é multi-dataset: o modelo de blocos é o hub, e você o compara contra quantos outros conjuntos precisar — as compostas por trás dele, uma campanha de adensamento, uma re-estimativa, um segundo modelo. Cada conjunto é um par, com seu próprio painel de importação, filtros, colunas calculadas e (se tiver coordenadas) geometria. O endereço é https://endarthur.github.io/bma/ — e também funciona offline depois da primeira visita (ver §17).

2Primeiros passos

2.1 O dataset de exemplo

Na tela inicial há um botão “⤓ Download example dataset”. Ele gera, no seu próprio navegador, um bma-example.zip contendo um modelo de blocos sintético, as compostas de sondagem “por trás” dele, as tabelas brutas de sondagem (collar/survey/assays, para o §13) e um TUTORIAL.txt passo a passo. É a melhor forma de conhecer a ferramenta: os dados foram desenhados para que cada gráfico mostre algo interessante (há um viés de +2% de Fe plantado nas amostras, de propósito).

O zip de exemplo é também um projeto empacotado: se você arrastar o próprio zip de volta para o BMA e confirmar, tudo abre já configurado (ver §15).

2.2 Carregando seus dados

Arraste o arquivo para a zona pontilhada ou clique nela para procurar. Formatos aceitos:

  • .csv, .txt, .dat — texto delimitado (vírgula, ponto-e-vírgula, tab ou espaço, detectado automaticamente);
  • .zip — com um ou mais CSVs dentro (você escolhe qual analisar);
  • .bma.json / .bma.zip — projetos salvos ou empacotados (§15).

Arquivos abertos recentemente aparecem como cartões na tela inicial — em navegadores baseados em Chromium, clicar reabre direto, sem procurar de novo.

Você não precisa começar por um modelo de blocos. “Começar um projeto vazio” abre o workspace sem modelo nenhum — útil quando você só tem furos de sondagem ou dados de pontos dispersos. A partir daí você adiciona conjuntos (próximo parágrafo), e as abas que precisam de um modelo (curva de teor–tonelagem, a referência Δ% do modelo) simplesmente miram o conjunto que você escolher. Um projeto sem modelo se salva como qualquer outro e volta pela lista Projetos na tela inicial (ele não tem um arquivo de modelo para arrastar, então essa lista é como você o reabre); você também pode Empacotar (Pack) num único .bma.zip (§15). Você pode dar um modelo ao projeto a qualquer momento — Carregar arquivo de modelo… na aba Importar, ou arrastando um CSV de modelo — sem perder os conjuntos que você já adicionou.

Adicionando conjuntos. Com um projeto aberto, use ＋ Adicionar conjunto no pé da árvore de dados (§14), o lançador ＋ na barra de abas, ou Dados → Adicionar conjunto de pontos / de furos. Cada novo conjunto ganha seu próprio painel de importação; a primeira comparação é rotulada d2, a seguinte d3, e assim por diante. Você também pode simplesmente arrastar um arquivo para um projeto aberto: o BMA pergunta se deve adicioná-lo como conjunto de comparação ou usá-lo como modelo (substituindo o atual) — ele nunca substitui seus dados em silêncio.

2.3 Ajuda contextual

Em qualquer aba, pressione F1 (ou clique no ? da barra) para abrir a ajuda daquela aba, com a explicação de cada controle.

2.4 A área de trabalho: painéis que convivem

Em telas largas, as abas do BMA são painéis acopláveis. Tudo se comporta como uma barra de abas comum — até você arrastar uma aba:

  • solte no vão entre painéis (ou nas bordas) para dividir a área — Statistics ao lado do Swath, GT embaixo do Categories;
  • solte sobre o corpo de um painel para destacá-la em uma janela flutuante, que você move e redimensiona à vontade; solte sobre uma barra de abas para reacoplar;
  • botão direito em uma aba: Float, Duplicate, Move to new rail, Close, Close others; botão direito na barra de título de uma janela flutuante: Dock / Close.

Os painéis de análise também são clonáveis. Use ＋ na barra de abas (ou botão direito numa aba → Duplicate) para abrir um segundo Statistics, Categories, Swath, StatsCat, GT ou Export — uma visão independente, com sua própria seleção, direções, referência e conjunto-alvo. Coloque as estatísticas do Fe ao lado das do SiO₂, dois swaths em eixos diferentes, ou a curva teor–tonelagem do modelo ao lado da de um conjunto de comparação, lado a lado. Clones de Statistics, Categories e StatsCat são visões independentes da mesma análise; clones de Swath e GT rodam sua própria passada no worker; clones de Export exportam cada um o seu conjunto. Uma duplicata leva a configuração do painel de origem; cada clone é salvo com o projeto.

Um workspace novo abre um conjunto enxuto — Import Model, Statistics e Categories — e os demais painéis abrem sob demanda, então você começa com uma superfície limpa em vez de uma parede de abas. Fechar um painel também não perde nada — o conteúdo continua vivo. Para abrir ou reabrir qualquer painel, use ⋮ → Panels (os abertos aparecem marcados), a árvore de dados (§14) ou clique com o botão direito no fundo de qualquer barra de abas. ⋮ → Reset layout volta a esse arranjo padrão de pilha única. O arranjo é salvo junto com o projeto, e a árvore de dados vive em uma coluna própria à esquerda, recolhível pelos botões ◀/▶.

Em telas estreitas (celular), nada disso muda: o BMA mantém a navegação clássica por abas.

Abrindo um painel. Um workspace novo mostra só Import Model, Statistics e Categories. As outras abas (Calc, GT, Swath, StatsCat, Export) abrem sob demanda — por View → Panels (um ✓ marca as abertas), pela árvore de dados, ou com o botão direito numa barra de abas. O passo a passo abaixo diz “abra a aba GT” etc.; é assim que se faz. (Um projeto empacotado ou salvo restaura o layout que você deixou, então o projeto de exemplo abre com tudo já no lugar.)

2.5 Sua primeira sessão: validando o modelo de exemplo

Este é o jeito mais rápido de entender o BMA — uma validação completa modelo × amostras no exemplo embutido, de ponta a ponta, em uns quinze minutos. O exemplo foi construído com três defeitos plantados de propósito: as amostras carregam um viés de +2% de Fe no laboratório, uma campanha de adensamento agrupa um quarto dos furos no leste rico (um viés de amostragem) e o modelo de blocos é mais suave que as amostras pontuais. Você vai fazer os três se revelarem.

Preparo. Na tela inicial clique ⤓ Download example dataset, depois arraste o bma-example.zip resultante de volta ao BMA e confirme. Tudo abre pré-configurado. (Se preferir montar à mão para aprender os controles, siga os passos abaixo carregando o bma-example-model.csv primeiro; abra cada aba conforme chegar nela, conforme a nota acima.)

Passo 1 — Leia o modelo antes de confiar nele. No painel Import Model, após Analyze ▶, troque o painel direito para Summary. Confirme como é uma importação limpa: blocos de 10 m, grid 24×24×10, fill ratio 100% e o bloco de saúde de dados verde (nenhuma linha descartada). É a sua conferência de “o arquivo entrou certo” (§4).

Passo 2 — Olhe a distribuição de teor. Abra Statistics. Leia a média e o CV do Fe. Clique no nome Fe para plotar sua CDF abaixo, depois aperte Prob e ponha o eixo X em log: uma quase-reta significa aproximadamente lognormal — sua linha de base antes de qualquer comparação (§7.3).

Passo 3 — Traga as amostras. As amostras já vêm no projeto empacotado; senão adicione-as com Data ▸ Add point dataset e bma-example-samples.csv, ponha Weight = SUPPORT e Analyze (§12.1). De volta ao Statistics, aux:Fe aparece recuado sob Fe com uma linha Δ% — e ela lê bem acima de +2%. Esse é o destaque, e é grande demais de propósito: o agrupamento está inflando. Os passos 6–7 separam isso.

Passo 4 — Confirme, e descubra o tipo. Clique em aux:Fe para sobrepor sua CDF tracejada (a do modelo é mais íngreme — suporte). Depois troque o painel de CDF para Q–Q com Fe e aux:Fe selecionados: a nuvem corre paralela e acima da linha de 45°. Um deslocamento paralelo é viés aditivo, e a distância vertical é esse viés em % de Fe (§12.4).

Passo 5 — Veja que é geral, não local. Abra Swath, mantenha a direção X e Generate. A série tracejada das amostras cavalga acima da sólida do modelo em todas as bandas — um deslocamento espacialmente consistente, não um artefato de borda. Marque Y e Z também para as outras direções (§11, §12.3).

Passo 6 — Descarte o efeito da malha. No painel do conjunto de amostras, abra Declustering e Run declustering. A média ingênua (~54,3% Fe) cai para ~52,9% desagrupada — parte da riqueza aparente era só o adensamento no alto teor. Clique Use as weight e re-Analyze (§12.7).

Passo 7 — Tire a conclusão. Releia o Δ% sob o Fe agora que as amostras estão desagrupadas: ele cai para cerca dos +2% plantados. Esse resíduo é a parte que o desagrupamento não removeu — o viés de laboratório. Separar viés de amostragem (desagrupamento corrige) de viés de teor (não corrige) é o ponto inteiro do exercício, e o momento em que o propósito do BMA fica claro.

Passo 8 (opcional) — Leve uma decisão adiante. Troque o painel das amostras para Top-cut, arraste a linha vermelha do corte e use Copy calcol → cole no Calc → Analyze para aplicar um corte; ou use Export para gravar o conjunto limpo (§12.8, §16). Diagnósticos voltam para os dados.

Agora nos seus próprios dados, um modelo que passa é a imagem espelhada do exemplo: Δ% perto de zero em todas as métricas, CDFs diferindo só nas caudas (suporte), séries de swath se entrelaçando banda a banda, e o desagrupamento quase não mexendo na média das amostras. Onde seus dados se afastam disso é exatamente onde olhar.

3Importação — conferindo antes de analisar

Assim que o arquivo cai, o BMA lê as primeiras ~100 linhas e mostra o painel Import Model (o modelo é o primeiro conjunto de dados; conjuntos de comparação carregados depois ganham seus próprios painéis de importação, §12). Nada pesado rodou ainda — este é o momento de conferir:

  • Tipos — cada coluna é marcada NUM (numérica) ou CAT (categórica). Clique na etiqueta para forçar o tipo, ou desmarque Skip para excluir a coluna da análise.
  • XYZ — as colunas de coordenadas são detectadas pelo nome (X/EAST/LESTE…); corrija nos seletores se necessário. Sem XYZ não há geometria, swath nem GT por volume.
  • DXYZ — opcional: colunas de dimensão por bloco, para modelos sub-blocados.
  • Filtro por coluna — expressões rápidas tipo > 0 que pré-filtram valores daquela coluna (ex.: ignorar teores zero).
  • ZIP — se o arquivo for um zip com vários CSVs, um seletor de entrada aparece no topo.
  • Filtro de linhas — a caixa FILTER no topo deste painel pré-filtra as linhas do modelo antes da análise (§5).

Quando estiver satisfeito, clique Analyze ▶ no canto superior esquerdo.

4Análise e a visão Summary

A análise roda em streaming, com barra de progresso e botão de cancelar (cancelando, o último resultado completo é restaurado). O Summary não é uma aba separada — fica dentro do painel Import Model, que ganha um alternador Preview | Summary assim que existe uma análise (espelhando como cada dataset de comparação mostra seu próprio resumo, §12). Ao final da execução, o painel muda para a visão Summary, que mostra:

  • contagem de linhas, colunas, tempo de processamento e estimativa de memória dos resultados;
  • geometria do grid: tamanho de bloco em X/Y/Z, origem, extensão, ordenação das coordenadas, detecção de sub-blocos e fill ratio (fração do envelope realmente preenchida).

Os resultados ficam em cache local: reabrir o mesmo arquivo com a mesma configuração restaura tudo instantaneamente. Qualquer mudança de configuração (filtro, calcol, peso…) marca a análise como desatualizada — o botão Analyze fica em destaque pedindo um novo Execute.

5O filtro de linhas

A caixa FILTER fica no topo do painel Importar do modelo (§3). Aceita uma expressão JavaScript sobre o objeto de linha r; só as linhas em que a expressão é verdadeira entram em todas as estatísticas e gráficos do modelo. Exemplos:

r.Fe > 55
r.LITO === 'HEM' && r.Fe > 60
between(r.Z, 820, 860)
r.RATIO > 4        // filtros podem usar colunas calculadas!

Clique Apply (e depois Analyze) para aplicar; Clear remove. O Apêndice A lista todas as funções auxiliares disponíveis (clamp, ifnull, between…). Há autocompletar de nomes de colunas enquanto você digita.

Básico de expressões (a mesma sintaxe move filtros, filtros locais e calcols — você só precisa de algumas peças):
  • r.Fe lê a coluna Fe da linha atual r. Coluna com espaços ou símbolos no nome: r["Cu (ppm)"].
  • Comparar com > < >= <=; igual é === e diferente é !== (dois/três caracteres — um único = atribui, não testa).
  • Combinar condições com && (e), || (ou), ! (não): r.Fe > 55 && r.LITO === 'HEM'.
  • Texto vai entre aspas: 'HEM' ou "HEM". Números não.
  • Se/senão numa linha usa ? : — condição ? valorSeVerdadeiro : valorSeFalso (usado nas calcols, §6).
Isso basta para quase tudo; o Apêndice A acrescenta funções prontas (between, cap, clamp…) para os casos restantes.

Na aba Categories, desmarcar valores na tabela monta esse filtro automaticamente para você (§8).

6Calc — colunas calculadas

A aba Calc cria novas variáveis a partir das existentes. Você escreve um pequeno bloco de código que adiciona colunas a cada linha — atribua a um nome novo no objeto de linha r e esse nome vira uma coluna:

r.RATIO = r.Fe / r.SiO2;
r.ORE = r.Fe > 55 ? 'ore' : 'waste';
  • Simulate roda o código nas linhas de amostra e mostra as colunas novas detectadas, com tipo inferido (NUM/CAT) e uma tabela de preview. Erros aparecem na hora.
  • O navegador de variáveis (esquerda) insere r.coluna com um clique; expandir uma variável mostra min/máx/média ou os valores categóricos.
  • Vale JavaScript completo: condicionais, laços, strings (r.LITO.startsWith('BIF')).
  • O BMA infere o tipo de cada coluna nova pelos valores. Para forçar (avançado), some o nome à lista de tipo: r.META.num.push('nome') marca como numérica, r.META.cat.push('nome') como categórica.
  • Depois de simular, clique Analyze — as calcols são computadas durante a análise e aparecem em todas as abas (com etiqueta CALC), inclusive como candidatas a peso, eixo de GT, variável de swath etc.

Calcols posteriores podem usar anteriores (r.B = r.A * 2), mas a ordem no código é a ordem de avaliação. Evite nomear colunas com nomes de funções (abs, min…), que seriam sombreados.

Quando há mais de um conjunto carregado, um seletor de conjunto no topo do editor escolhe a qual dataset as calcols se aplicam — o modelo ou qualquer conjunto de comparação (cada um guarda seu próprio bloco de código); ver §12.6.

7Statistics — estatísticas, pesos e CDF

Com dois ou mais conjuntos analisados, um seletor Dataset aparece no topo da lateral: ele escolhe qual conjunto é o primário do painel (suas colunas são as linhas da tabela), e todo outro conjunto — o modelo incluído — vira uma comparação Δ% abaixo. Por padrão o primário é o modelo, então um projeto de modelo único fica exatamente como descrito aqui; o seletor só significa que qualquer conjunto pode ser a âncora (§12.2).

7.1 A tabela

Uma linha por variável numérica; colunas configuráveis na barra lateral: contagem, nulos, zeros, mín/máx, média, desvio, CV%, assimetria, curtose e os percentis que você quiser (presets de quartis/decis ou lista própria, ex.: 5,10,25,50,75,90,95). Copy table copia tudo como tabela para colar no Excel.

7.2 Pesos

O seletor Weight pondera todas as estatísticas numéricas por uma coluna (ou calcol) — por exemplo, proporção de minério ou densidade (média ponderada por massa). O peso é um papel da variável: o mesmo vale para o Swath, e aparece como etiqueta W na árvore de dados (§14). Detalhes que importam:

  • mudar o peso exige um novo Analyze;
  • linhas com peso ausente, zero ou negativo são excluídas (o contador aparece sob o seletor);
  • com peso ativo, desvio/assimetria/curtose usam a forma populacional (correções de amostra pequena não se aplicam a pesos de escala arbitrária);
  • ative a métrica ΣW para ver o total de peso ao lado da contagem;
  • os percentis e a CDF também ficam ponderados — peso 2 equivale exatamente a repetir a linha duas vezes.

7.3 CDF, Prob e Q–Q

Clique no nome de qualquer variável na tabela para adicionar sua curva de distribuição acumulada (CDF) ao painel inferior; clique de novo para remover. Escala linear ou log, tooltip com leitura por curva, e botões SVG/PNG para exportar o gráfico. O modo Q–Q é descrito em §12.4 — ele compara duas distribuições quantil a quantil.

O botão Prob troca o eixo Y para a escala de probabilidade normal (probit), com réguas de 0,2% a 99,8% — o clássico papel log-probabilístico da geoestatística. A leitura:

  • com X em log, uma distribuição lognormal plota como reta: curvatura é desvio de lognormalidade, quebras de inclinação sugerem populações misturadas;
  • as caudas — exatamente onde decisões de capping e de alto teor são tomadas — ficam expandidas em vez de coladas no 0% e no 100%;
  • funciona com quantas curvas você quiser, inclusive as do auxiliar (tracejadas), para comparar caudas modelo × amostras.

Lendo nos seus próprios dados: uma CDF do modelo logo dentro da das amostras (um pouco menos espalhada, mesmo centro) é a assinatura normal do suporte de bloco. Curvas que se cruzam, ou diferem no corpo e não só nas caudas, apontam uma diferença real de distribuição — domínio, unidades ou estimativa — não apenas suporte.

8Categories — variáveis categóricas

A aba Categories explora colunas categóricas (litologia, domínio, classificação…). Selecione uma coluna na lateral para ver:

  • gráfico de barras com contagens, percentuais e linha de Pareto (com marca dos 80%);
  • entropia de Shannon (H) como medida de mistura;
  • tabela de valores com busca, contagem e percentual.

Recursos que valem conhecer:

  • Cores — clique no quadradinho de cor de um valor para personalizá-la; a cor escolhida vale em todas as abas (StatsCat, GT…).
  • Ordem custom — no modo de ordenação Custom, arraste as linhas da tabela; a ordem é respeitada nas outras abas.
  • Filtro por checkbox — desmarcar valores monta a expressão e oferece aplicá-la ao filtro de linhas do modelo (uma barra Aplicar aparece; ex.: esconder o estéril de todas as estatísticas).

Com um conjunto de comparação analisado, esta aba ganha a comparação de proporções modelo × amostras — ver §12.5 (a captura está lá). O Categories também é clonável (§2.4).

Como no Statistics, com 2+ conjuntos analisados um seletor Dataset escolhe o conjunto categórico primário do painel (as barras); os outros se sobrepõem como losangos de proporção. O modelo é o primário padrão.

9StatsCat — estatísticas por grupo

O StatsCat quebra as estatísticas numéricas por uma variável categórica (Group By) — média de Fe por litologia, por exemplo. Escolha o grupo e as variáveis na lateral (pode exigir um novo Analyze para incluir variáveis ainda não agrupadas) e marque quais grupos exibir.

  • Tabela por grupo — contagem, média, desvio e percentis para cada combinação variável × grupo;
  • CDF por grupo — curvas coloridas pela categoria (com as cores da aba Categories), escala linear/log e intervalo manual;
  • Cross-tab — matriz de contagem (ou % por linha/coluna) entre duas categóricas.

Se um peso estiver ativo na aba Statistics, as estatísticas agrupadas também ficam ponderadas (mesma análise).

Com 2+ conjuntos analisados, um seletor Dataset escolhe qual conjunto o StatsCat quebra (o modelo por padrão) — assim você pode agrupar os teores de um conjunto de comparação pelas categorias dele.

10GT — curvas teor–tonelagem

A aba GT varre teores de corte e produz as curvas clássicas de tonelagem, teor médio acima do corte e conteúdo metálico. Configuração:

  • Variáveis de teor — uma curva por variável marcada, cada uma com sua unidade (%, g/t, ppm…), que controla o fator do metal;
  • Densidade — uma coluna, um valor constante (ex.: 2,8 t/m³) ou nenhuma. Sem densidade, a “tonelagem” é na verdade volume (m³) — a interface avisa;
  • Peso — coluna multiplicadora opcional por linha. Quando o conjunto não é um grid (sem volume de bloco), ela serve também como coluna de tonelagem/volume: escolha-a para ponderar a tonelagem por um valor explícito por linha;
  • Volume do bloco — detectado automaticamente: DXYZ por linha se houver, senão a geometria do grid — mas só quando o conjunto é classificado como grid (§3). Um modelo classificado como ponto, ou sem DXYZ/override, cai para tonelagem por contagem (1 por linha); a lateral avisa e diz como obter ponderação por volume (defina como grid, atribua DXYZ, ponha um Override ou escolha uma coluna no Peso). O volume da geometria sempre pode ser sobrescrito;
  • Cutoffs — intervalo (mín/máx/passo) ou lista própria;
  • Group by — curvas separadas por categoria (litologia na figura), com seleção de quais grupos mostrar e linha de total;
  • Unidade de tonelagem — t, kt, Mt ou divisor próprio.

Passe o mouse para ler valores exatos no crosshair. Cada gráfico tem Copy SVG e Download PNG (versão clara, em alta resolução, com o nome da variável no título), e uma tabela recolhível com botão Copy.

10.1 Curva teórica a partir das amostras

A seção Theoretical (samples) da lateral sobrepõe ao gráfico a curva teor–tonelagem teórica derivada de uma distribuição de amostras, com correção de suporte por afinidade (correção afim): a variância da distribuição pontual é reduzida pelo fator f (slider), simulando o suporte de bloco. Está disponível sempre que o GT mira um dataset em grid — o modelo por padrão, ou qualquer estimativa de comparação em grid escolhida no seletor Dataset do GT — com um seletor from escolhendo qual dataset carregado fornece as amostras (você valida uma estimativa contra as amostras das quais ela foi construída, nunca contra ela mesma). Pontos de referência:

  • f = 1 reproduz exatamente a GT empírica das amostras (sem correção);
  • f < 1 encolhe a variância em torno da média — menos tonelagem acima de cortes altos, teores médios mais baixos acima do corte: o efeito esperado ao passar de amostras pontuais para blocos;
  • a curva usa o peso ativo do auxiliar (ex.: desagrupamento, §12.7) — compare distribuições representativas, não enviesadas pela malha;
  • a sobreposição aparece nos gráficos sem group-by, escalada para a tonelagem total da estimativa-alvo.

Se a curva da estimativa cair razoavelmente sobre a teórica para um f plausível (tipicamente 0,4–0,8 conforme a razão de suportes), a suavização da estimativa é compatível com a mudança de suporte esperada. Uma estimativa muito mais “suave” que a teórica em qualquer f é sinal de sobre-suavização — teores altos diluídos demais.

11Swath — perfis direcionais

O Swath fatia o modelo em bandas ao longo de uma direção e mostra como cada variável se comporta no espaço — a ferramenta padrão para enxergar tendências e validar estimativas.

  • Direções — marque X (O→L), Y (S→N) e/ou Z (base→topo): todas as direções marcadas são calculadas em uma única passada e viram guias no resultado. Cada direção tem seu bin width (padrão = tamanho do bloco naquele eixo; aumente para suavizar);
  • Custom — direção rotacionada definida por dip direction (0° = norte, horário), dip e rake, gerando o sistema U/V/W (U = direção do rake no plano, W = polo). Para o caso clássico “azimute + mergulho”, use dip direction = azimute, dip = mergulho, rake = 90°;
  • Estatística — média ± desvio, P25/50/75 ou P10/50/90 (faixas sombreadas);
  • Weight — pondera as estatísticas das bandas do conjunto primário (rotulado Weight (model) quando o primário é o modelo, onde é o mesmo peso da aba Statistics — mudar num lugar muda no outro);
  • Display — liga/desliga faixas, barras de contagem e tabela; escala Y linear/log; layout Overlay (tudo num gráfico, um eixo Y por variável) ou Split (um painel por variável);
  • Cores — clique no quadradinho ao lado da variável.
  • Dataset — com 2+ conjuntos analisados, escolhe o conjunto primário do painel (a série sólida); todo outro conjunto se sobrepõe tracejado. O modelo é o primário padrão.

Crosshair com leitura por variável, Copy SVG / Download PNG, e tabela por banda com cópia. A captura desta aba está em §12.3, já com a sobreposição das amostras — o uso mais valioso do swath.

12Conjuntos de comparação — validação

O BMA compara o modelo de blocos contra um ou mais outros conjuntos de dados — tipicamente as compostas de sondagem que alimentaram a estimativa, mas também uma campanha de adensamento, uma re-estimativa ou um segundo modelo. É a resposta da ferramenta à pergunta central da validação: o modelo honra os dados?

Cada conjunto de comparação é um par (peer) com seu próprio painel de importação. Adicione um por Data ▸ Add point dataset… ou Add drillhole set… (cada um abre uma nova aba de importação), ou arraste um arquivo para uma nova aba de conjunto. O primeiro comparativo é historicamente rotulado aux; os seguintes são d2, d3… Cada um carrega suas próprias coordenadas, filtro, colunas calculadas, peso e — quando tem XYZ — sua própria geometria.

Um conjunto de comparação é um conjunto separado de linhas, não colunas extras no modelo: não há join por bloco. A comparação é estatística (médias, distribuições, proporções) e espacial (swath). Mas, no mais, um conjunto de comparação é de primeira classe — se traz coordenadas, ganha geometria de grid, override grid/point, bounding box e exportação OBJ + CSV pela sua visão Summary (§16), exatamente como o modelo.

12.1 Carregando e configurando

  • Carregar — arraste o arquivo para o painel de importação do conjunto, ou: se o dado principal veio de um zip com vários CSVs, use “use an entry from…” para apontar outra entrada; se você arrastar um zip aqui, um seletor de entrada aparece na lateral.
  • Display prefix — apenas o rótulo das variáveis do conjunto na interface (aux:Fe, samples:Fe…). Cosmético: nos códigos e filtros a referência é sempre o handle do conjunto (aux., d2.…).
  • Coordinates — X/Y/Z do conjunto (detectadas por nome: EAST/NORTH/ELEV…). Modelo e comparação precisam estar no mesmo sistema de coordenadas para o swath fazer sentido.
  • Block geometry — o mesmo override grid/point/auto do modelo (§3); um conjunto de amostras esparso fica em auto e é detectado como pontos.
  • Filter — pré-filtra as linhas do conjunto, com o handle dele: ex. aux.SUPPORT > 1.
  • Weight — peso para todas as estatísticas do conjunto: comprimento de composto (na figura, SUPPORT), uma coluna ou calcol qualquer, ou (declustered weights) — os pesos de desagrupamento calculados pelo próprio BMA (§12.7). Pesos ausentes/≤0 excluem a linha.
  • Analyze — roda a passada estatística completa do conjunto. Necessário para as comparações de Statistics/CDF/Q–Q/Categories (o swath roda a própria passada e não depende dele). Mudou filtro/peso/calc? O status avisa “re-run Analyze”.

12.2 Statistics e CDF comparadas

Depois do Analyze de um conjunto de comparação, suas variáveis aparecem ao final da lateral (pré-marcadas quando têm homônima no modelo). Na tabela, aux:Fe fica logo abaixo de Fe — média contra média, CV contra CV — com uma linha Δ% mostrando a diferença contra o conjunto de referência. No exemplo, a média de Fe das amostras roda ~2% acima da do modelo: o viés plantado, visível de imediato. Clique no nome aux:Fe para sobrepor sua CDF tracejada: a curva do modelo é mais íngreme — o suporte de bloco suaviza as caudas que as amostras pontuais ainda têm (o “efeito de suavização” da krigagem).

Lendo nos seus próprios dados: Δ% dentro de um ou dois por cento em todas as métricas significa que o modelo honra as amostras no agregado. Um sinal consistente nas médias de vários teores (todos modelo-baixo ou todos modelo-alto) sugere um viés genuíno; uma diferença grande de CV ou assimetria com médias parecidas costuma ser suporte (média de bloco) — confirme com o Q–Q (§12.4) e uma comparação desagrupada (§12.7) antes de concluir.

Dois controles deixam a comparação flexível. Um seletor Dataset (topo da lateral, mostrado com 2+ conjuntos analisados) define o primário do painel — o conjunto cujas colunas são as linhas; todo outro conjunto, o modelo incluído, vira comparação. O primário padrão é o modelo, então nada muda numa checagem comum modelo × amostras; o seletor só deixa você inverter a âncora — tornar as amostras o primário e ler o modelo como comparação, por exemplo. Depois, com três ou mais conjuntos carregados, surge uma linha de chips de conjunto: alterne quais comparações aparecem e escolha a referência do Δ% (o denominador) por painel — assim um clone de Statistics pode comparar contra as amostras enquanto outro compara contra uma re-estimativa.

12.3 Swath comparado

Na aba Swath, as variáveis de cada conjunto de comparação entram no fim da lista (divididas por um separador com o nome do conjunto). A série comparativa é tracejada na mesma cor e mesmo eixo Y da variável homônima do modelo — de propósito: eixos independentes “autoescalariam” a diferença para fora do gráfico, escondendo exatamente o viés que o gráfico existe para mostrar. O tooltip traz n= por banda. Todo conjunto carregado é processado na mesma passada; os chips de conjunto escolhem quais desenhar.

Lendo nos seus próprios dados: uma sobreposição saudável tem a série tracejada e a sólida se entrelaçando banda a banda. Uma diferença persistente para um lado na maioria das bandas é viés espacial (condicional); divergência só nas bandas extremas — onde a contagem de amostras é minúscula (olhe as barras de contagem) — costuma ser só suporte fino ali, não falha do modelo.

12.4 Q–Q: o viés medido

No painel de CDF, o botão Q–Q troca para o gráfico quantil–quantil: a primeira curva selecionada vira o eixo de referência (X) e as demais plotam seus quantis contra ela, de P1 a P99 (pontos maiores nos decis; passe o mouse para ler os valores). Leitura:

  • pontos sobre a linha tracejada de 45° → distribuições equivalentes;
  • deslocamento paralelo → viés (a distância vertical É o viés, em unidades da variável);
  • rotação/achatamento → diferença de dispersão — tipicamente o suporte de bloco encolhendo a variância.

Funciona entre quaisquer duas curvas (dois domínios do modelo, por exemplo), não só modelo × amostras. Escala log–log disponível; exporta com os mesmos botões SVG/PNG.

Lendo nos seus próprios dados: pontos sobre a linha de 45° = as duas distribuições batem. Um deslocamento paralelo = viés aditivo (leia o tamanho no eixo). Uma rotação para fora da linha de 45° = uma diferença de variância (dispersão): a nuvem inclina para o conjunto mais espalhado — amostras pontuais são mais largas que estimativas em suporte de bloco, então uma leve inclinação é a assinatura normal do suporte, enquanto uma rotação forte é sobre-suavização. Uma dobra ou “S” = a concordância muda com o teor, muitas vezes uma questão de capping/top-cut em alto teor.

12.5 Categories comparado

Quando um conjunto de comparação tem uma coluna categórica de mesmo nome (LITO × LITO), a aba Categories sobrepõe um losango vazado por valor, marcando a proporção de cada conjunto na mesma escala das barras. A tabela ganha colunas n / % por conjunto, e valores que só existem em um comparativo aparecem ao final com a etiqueta comparison-only — uma litologia logada que o modelo nunca usou é exatamente o tipo de desacordo que vale enxergar.

12.6 Calcols no auxiliar

O seletor de conjunto no editor Calc aponta o editor para qualquer conjunto de comparação: as colunas completam como aux.coluna (ou d2.coluna…), o preview usa as linhas daquele conjunto e as colunas criadas valem para tudo que é dele — inclusive como peso e nas comparações. Se a calcol auxiliar tiver o mesmo nome de uma do modelo (ex.: RATIO nos dois), elas se pareiam automaticamente em todas as comparações (e o pareamento pode ser editado à mão — §14). Depois de editar, rode o Analyze do conjunto de novo.

12.7 Desagrupamento (declustering)

Sondagem adensa onde o teor é alto — e a média “ingênua” das amostras herda esse otimismo. A seção Declustering da lateral implementa o desagrupamento por células do GSLIB (DECLUS): o espaço é dividido em células, cada amostra recebe peso inverso à lotação da sua célula, e uma varredura de tamanhos de célula procura o que minimiza (ou maximiza) a média desagrupada.

  • Cell sizes — faixa da varredura (automática: do 1/100 ao 1/2 da maior extensão) e número de passos; anis Y/Z deforma as células para malhas anisotrópicas;
  • Find — minimizar (caso típico: adensamento em alto teor) ou maximizar a média desagrupada;
  • Run declustering — roda a varredura e mostra: média ingênua → média desagrupada, tamanho ótimo de célula, faixa de pesos e a curva interativa (crosshair para ler; clique para fixar um tamanho de célula específico — o pino sobrevive a re-runs do projeto);
  • Use as aux weight — aplica os pesos como peso do auxiliar; o seletor Weight passa a mostrar (declustered weights). Rode o Analyze auxiliar de novo: todas as comparações passam a usar a distribuição desagrupada.

No dataset de exemplo, a malha tem adensamento plantado na zona rica de propósito: a média ingênua das amostras (~54,3% Fe) cai para ~52,9% desagrupada — e a diferença restante contra o modelo é o viés de laboratório plantado. Separar viés de amostragem (desagrupamento resolve) de viés de teor (desagrupamento não resolve) é exatamente o que esta ferramenta faz.

Os pesos nunca são salvos no projeto — só os parâmetros. Ao reabrir, rode Run declustering de novo (o status avisa). Mudar filtro, calcols ou coordenadas do auxiliar invalida os pesos automaticamente.

12.8 Top-cut — análise de capping

O botão Top-cut (topo da área principal da aba Aux) troca a prévia pela análise de capping da variável escolhida — as quatro vistas clássicas para decidir teor de corte superior, todas sobre a distribuição exata das amostras:

  • histograma e gráfico log-probabilístico — onde a cauda descola da lognormal é o candidato natural a corte;
  • média capada × corte — quanto a média cede conforme o corte desce;
  • metal removido × corte — % do metal total cortado.

A linha vermelha é o corte: arraste-a em qualquer gráfico (ou digite o valor) e a faixa de status recalcula na hora — n capado, % das amostras, média e CV antes/depois, metal removido. Raw | Declustered alterna para a distribuição ponderada pelos pesos de desagrupamento (§12.7), e Linear | Log controla o eixo X.

Decidido o corte, Copy calcol copia a linha pronta — ex.: aux.Fe_cap = cap(aux.Fe, 63.91); — para colar no editor Calc (modo Aux). A variável capada entra nas comparações como qualquer outra. Para capar antes de uma limpeza (ifnull, filtros), aponte o top-cut para a calcol limpa em vez da coluna crua.

13Furos de sondagem — das tabelas brutas às compostas

Campanhas reais não chegam compostas: chegam como collar (boca do furo), survey (desvios) e intervalos (análises). A tela de importação de cada dataset tem um cartão Drillhole set que recebe os três arquivos (ou um zip com eles): cada um cai no seu papel — detectado pelo cabeçalho — e o mapeamento de colunas aparece preenchido, sempre editável. Como cada dataset tem o seu cartão, um projeto pode ter vários conjuntos de sondagem. Cada tabela tem ainda um ↓ CSV no título, e Download tables (zip) empacota o conjunto bruto exatamente como foi importado.

13.1 A superfície de composição

O cartão é uma pequena bancada de composição. Acima das opções de geometria:

  • Convenção de dip — a linha destacada em âmbar: o BMA detecta pelo levantamento se o arquivo usa dip positivo ou negativo para baixo, e o botão está ali para corrigir. Sempre confira contra a exportação dos seus surveys.
  • Método de desurvey — curvatura mínima (padrão), tangencial balanceado ou tangencial (segmentos retos — útil para reproduzir desurveys históricos);
  • Composite length — padrão = o comprimento de intervalo mais comum do arquivo; Min coverage filtra compostas pouco amostradas (desligado por padrão — nada é descartado em silêncio, a cauda curta é contada).

E os três controles que moldam como os intervalos viram compostas:

  • Break on (splits) — marque quaisquer colunas categóricas; a composta reinicia em cada contato dessas colunas, então nenhuma composta atravessa um contato de litologia ou zona. Marque mais de uma e ela quebra onde qualquer uma muda — sem precisar montar à mão uma chave de domínio combinada. Cada coluna de split entra na composta sem alteração (é constante dentro da composta por construção).
  • Density (mass-weight) — indique uma coluna de densidade e as médias de teor passam a ser ponderadas por comprimento × densidade (massa) — a média geologicamente correta quando há densidade. A própria coluna de densidade segue ponderada por comprimento, o SUPPORT segue o comprimento coberto, e intervalos sem densidade utilizável são excluídos da ponderação por massa e contados.
  • Combine rules — por coluna de saída, como ela agrega na composta: numérica mean (padrão, ponderada por comprimento/massa), sum, min ou max; categórica majority (por comprimento coberto) ou first (a mais rasa). Padrões sensatos por tipo, sobreponíveis por coluna.

Cada tabela de intervalos carrega também os seus próprios calcols & filter (a caixa Interval calcols & filter), igual a um dataset (§12.6): uma coluna derivada ou um filtro de linhas escrito aqui se aplica àquela tabela na exportação e na composição — o lar natural que uma única composta achatada nunca teve.

13.2 Mesclando uma segunda tabela de intervalos

Análises e litologia normalmente vivem em tabelas separadas, com quebras que não coincidem. Solte um segundo arquivo de intervalos e ele entra como tabela de mescla: o BMA o junta à primeira por BHID + FROM/TO via re-segmentação por união — cada segmento de saída cai dentro de no máximo um intervalo de cada tabela, então as colunas são carregadas sem palpite de agregação. A composta passa a carregar as colunas das duas tabelas (teor e domínio). Um segmento sem correspondente de um lado é preenchido com nulo e contado — nunca descartado em silêncio — e a tabela de mescla tem os seus próprios calcols/filtro pré-mescla (derive um domínio simplificado, ou descarte linhas marcadas no QA, antes da junção).

13.3 Composição & o resultado

Composite & load roda o desurvey (interpolação no arco), faz a composição ponderada (por comprimento ou massa) e carrega o resultado como aquele dataset: XYZ no ponto médio de cada composta, uma coluna SUPPORT (comprimento coberto) já aplicada como peso, e todas as outras colunas junto. Tudo das seções 12.1–12.8 funciona igual a partir daí.

O relatório de consistência lista tudo que não casou: BHIDs órfãos, FROM ≥ TO, intervalos sobrepostos, furos sem survey (dessurveyados verticais, contados), lacunas de mescla… nada some em silêncio. Abaixo dele, um histograma de distribuição de comprimento mostra o suporte antes (os comprimentos brutos dos intervalos) contra depois (o SUPPORT das compostas), no mesmo eixo — você vê num relance como a composição regularizou o suporte e onde fica a cauda curta de baixa cobertura (a população que o Min coverage mira). A faixa de proveniência guarda o Report e o Edit & re-composite.

O Pack (§15) de um projeto com sondagem carrega os arquivos brutos de collar/survey/intervalos (e qualquer tabela de mescla) + a receita de cada um — as compostas se re-derivam ao abrir, nunca são armazenadas (mesma filosofia dos pesos de desagrupamento). O dataset de exemplo traz um conjunto: bma-example-collar/survey/assays.csv, com dips negativos de propósito.

14A árvore de dados e os menus de contexto

O painel à esquerda das abas é a árvore de dados — o catálogo de propriedades de tudo que está carregado. Uma caixa de busca no topo filtra a árvore por nome de variável conforme você digita. Para cada dataset (modelo e auxiliar), as variáveis aparecem agrupadas em Coordinates (com os eixos X/Y/Z atribuídos), Grades, Categories e Calculated, cada uma com:

  • chip de cor — a cor da série nos gráficos (Swath, CDF…). Variável auxiliar pareada herda a cor da homônima do modelo, de propósito;
  • unidade — %, g/t, ppm… Uma unidade por variável, válida em todas as abas (mudar aqui muda no Swath, no GT e no Summary ao mesmo tempo);
  • papéis — etiquetas de função: W (peso das estatísticas), densidade e fator de tonelagem do GT;
  • pareamento (variáveis auxiliares) — ⇄ Fe indica com qual variável do modelo ela é comparada; ⇄ — em vermelho marca uma órfã, que não entra em comparação nenhuma.

Clique em qualquer variável para abrir o editor: cor (paleta ou hex, com “auto” para voltar ao padrão), unidade, papéis e — nas auxiliares — o seletor Paired with (model). O pareamento automático é por nome igual (sem diferenciar maiúsculas); aqui você corrige os casos em que os nomes diferem: FE_PCT das amostras pareado com Fe do modelo, sem precisar renomear nada. Tudo que você definir fica salvo no projeto.

A coluna da árvore recolhe e reabre pelos botões ◀/▶ no seu topo (em telas estreitas, pelo botão no canto esquerdo da barra de abas).

Botão direito sobre uma variável — em qualquer lugar: árvore, tabela ou lateral do Statistics, listas do Swath e do GT, colunas do Categories — abre o menu de contexto com as mesmas ações: Properties… (o editor acima, no lugar do clique), definir/limpar papéis, adicionar/remover curva CDF, desparear ou trocar o pareamento. Onde houver tabela com botão de cópia, o menu oferece Copy table; no painel de CDF, exportar SVG/PNG; nos valores do Categories, Set color…. Ctrl+botão direito sempre abre o menu nativo do navegador.

15Projetos: salvar, empacotar, compartilhar

15.1 Salvamento automático

Toda a configuração — preflight, filtros, calcols (de cada conjunto), seleções e clones de cada painel, pesos, cores, o layout do workspace — é salva automaticamente por arquivo no próprio navegador. Feche, reabra, arraste o mesmo arquivo: tudo volta como estava, incluindo a análise (que fica em cache). Um projeto sem modelo (§2.2) é salvo do mesmo jeito, sob sua própria identidade, e reabre pela lista Projetos na tela inicial, em vez de arrastar um arquivo de modelo; seus conjuntos depois aguardam o reenvio (rearraste cada um, ou use o Pack para carregá-los).

15.2 Save — o arquivo de projeto

O botão Save da barra baixa um nome.bma.json com a configuração completa (sem os dados). Para usar: arraste o .bma.json na tela inicial — o BMA pede o arquivo de dados correspondente; arraste-o em seguida e o projeto se aplica.

15.3 Pack — o projeto portátil

O botão Pack empacota dados + auxiliar + configuração em um único nome.bma.zip — o jeito certo de mandar uma análise inteira para um colega. No diálogo:

  • Title — nome do projeto (aparece na barra ao lado do arquivo e batiza o zip);
  • Contents — o modelo (se houver) e a configuração sempre vão; os conjuntos de comparação são opcionais (uma caixa cobre todos). Um conjunto derivado de furos empacota os arquivos brutos de collar/survey/intervalos, não o composto — eles se re-derivam ao carregar. Um projeto sem modelo é empacotado igual, sem entrada de modelo;
  • Compress — liga a compressão deflate (CSV encolhe 4–6×). Vem ligada para dados de até ~500 MB; acima disso o padrão é desligada, porque sem compressão o empacotamento e a reabertura têm custo de memória zero em qualquer tamanho. Não há limite de 4 GB: os arquivos usam Zip64 automaticamente quando preciso.

Quem recebe só precisa arrastar o zip no BMA: o diálogo acima pergunta se quer carregar o projeto embutido; confirmando, modelo e amostras saem do próprio arquivo e tudo abre configurado. (Open as zip ignora o projeto e abre como um zip comum.)

16Export — exportando CSV

A aba Export regrava um conjunto (já com o seu filtro aplicado e com as colunas calculadas disponíveis) como um novo CSV, também em streaming — funciona para arquivos de qualquer tamanho. Um seletor de Conjunto no topo escolhe qual exportar quando há mais de um carregado (o modelo, por padrão).

  • marque/desmarque colunas; arraste para reordenar; clique no nome de saída para renomear;
  • opções: delimitador, fim de linha, separador decimal, texto para nulos e precisão numérica;
  • barra de progresso com contagem de linhas durante a gravação.

Um seletor Dataset no topo da lateral (quando há dois ou mais conjuntos analisados) aponta o Export para qualquer conjunto — cada um com sua própria seleção de colunas. Um conjunto de comparação também pode ser exportado pela sua visão Summary: Export rows (CSV) grava todas as suas colunas e calcols (com o filtro do conjunto aplicado), e — quando ele é um grid — a bounding box sai como malha OBJ. Pontos e conjuntos derivados de sondagem emitem as mesmas formas que o modelo.

Além do CSV, espalhados pela interface há exports de gráfico (Copy SVG, Download PNG em Statistics/GT/Swath) e botões Copy de tabela (Statistics, GT, Swath, Categories) que colam direto em planilhas.

17Instalação, offline e atualizações

O BMA pode ser usado de três formas, do mais ao menos conectado:

17.1 No navegador ou instalado (recomendado)

  • Instalar como aplicativo — o BMA é um PWA: no Chrome/Edge, use o ícone de instalação na barra de endereço. Ele vira um app com janela própria e funciona offline depois da primeira visita.
  • Selo de build — o código no canto inferior direito (ex.: c18b3dc) identifica exatamente a versão que está rodando. Clique nele para verificar atualizações na hora.
  • Banner de atualização — quando uma versão nova é publicada, o BMA percebe sozinho (na carga seguinte) e mostra o banner acima; clique Reload para atualizar. Nenhum dado seu é afetado — projetos e caches são preservados.

17.2 Cópia local: um arquivo só

O aplicativo inteiro é um único arquivo HTML. Você pode salvá-lo (Ctrl+S na página, ou recebê-lo de alguém), renomear como quiser — bma.html, bma-c18b3dc.html — e abrir com clique duplo, sem internet, sem instalação e sem servidor. Tudo funciona: todas as abas, o dataset auxiliar, pesos, projetos empacotados, o exemplo embutido. O salvamento automático de configurações também funciona na cópia local.

Caveats da cópia local:

  • Ela nunca se atualiza — é uma fotografia da versão em que foi salva. O selo de build no canto diz qual: compare com o do site para saber se a sua cópia está atrasada (clicar no selo mostra “n/a”, já que não há de onde verificar).
  • A instalação como PWA e o modo offline automático são exclusivos da versão hospedada (um arquivo local já é offline por natureza).
  • As fontes da interface podem cair no monoespaçado do sistema se você estiver sem rede — só estética.

A cópia local é um ótimo plano B para ambientes sem internet ou com rede restrita — e, como nada nunca sai do navegador em nenhuma das formas, a privacidade é a mesma nas três.

17.3 Suporte de navegadores

O núcleo do BMA funciona em qualquer navegador moderno — análise, todos os gráficos e tabelas, conjuntos de comparação, empacotar e ler projetos .bma.zip. Algumas conveniências dependem da API File System Access, que hoje só os navegadores Chromium (Chrome, Edge, Brave, Opera) implementam; nos demais o BMA degrada com elegância.

DApêndice: receitas

Procedimentos curtos para os trabalhos comuns. Cada um aponta a seção com o detalhe.

Validar um modelo contra suas compostas

  1. Carregue o modelo, Analyze; confira o Summary (geometria, saúde) — §3, §4.
  2. Adicione as compostas como conjunto de comparação; ponha a coluna de comprimento como Weight; Analyze — §12.1.
  3. Statistics: leia o Δ% sob cada teor; sobreponha CDFs — §12.2.
  4. Q–Q do teor contra sua amostra: deslocamento paralelo = viés, inclinação = suporte — §12.4.
  5. Swath em X/Y/Z: confirme que o deslocamento é espacialmente consistente — §12.3.
  6. Desagrupe as compostas e re-Analyze: o Δ% residual é a parte que não é malha — §12.7.

Explorar um único modelo isolado (sem amostras)

  1. Carregue e Analyze; leia o Summary — §4.
  2. Statistics para cada teor (média/CV/percentis, CDF, Prob) — §7.
  3. Categories / StatsCat para domínios e teor-por-domínio — §8, §9.
  4. GT para o quadro teor–tonelagem; Swath para a tendência espacial — §10, §11.

Decidir e aplicar um top cut

  1. No painel do conjunto troque para Top-cut, Load distribution da variável — §12.8.
  2. Arraste a linha vermelha para onde a cauda deixa a lognormal no log-prob; observe o metal removido.
  3. Copy calcol, cole no Calc, Analyze — a variável capada flui para todas as vistas — §6.

Compor a partir de tabelas brutas de furos

  1. Solte collar + survey + assays (ou o zip deles) no cartão Drillhole set de um conjunto — §13.
  2. Confira papéis detectados, mapeamento de colunas e, sobretudo, a convenção de dip.
  3. Defina o comprimento de composta e Composite & load; leia o relatório de consistência.
  4. Analyze — as compostas se comportam como qualquer conjunto de comparação daí em diante.

Reconciliar duas estimativas

  1. Carregue o primeiro modelo; adicione o segundo como conjunto de comparação; Analyze os dois — §12.1.
  2. No Statistics, use o seletor Dataset para tornar qualquer um o primário; leia o Δ% com o outro como referência — §7, §12.2.
  3. Q–Q um contra o outro; Swath ambos para ver onde divergem no espaço — §12.3, §12.4.

Compartilhar uma análise inteira

  1. Pack → marque os conjuntos a incluir → grava um único .bma.zip — §15.3.
  2. Quem recebe arrasta o zip no BMA e confirma; tudo abre configurado.

EApêndice: galeria de interpretação

O dataset de exemplo ensina como é um problema (um viés plantado). Para ler o seu próprio modelo você também precisa saber como é o saudável. Cada par abaixo põe uma validação limpa (esquerda — modelo contra amostras sem viés e regularmente espaçadas) ao lado da falha do exemplo (direita), na mesma vista.

Tabela Δ% e CDF

Leia assim: Δ% perto de zero em toda métrica = concordância agregada. Um sinal não-nulo consistente nas médias é um viés a investigar (desagrupe primeiro, §12.7); uma diferença só no CV/dispersão com médias iguais costuma ser só suporte.

Q–Q

Leia assim: sobre a linha = batem; deslocamento paralelo = viés aditivo; uma inclinação para fora dos 45° = diferença de dispersão (suporte); uma dobra = desacordo que muda com o teor.

Swath

Leia assim: séries entrelaçadas = o modelo honra as amostras no espaço; uma diferença persistente para um lado = viés condicional; divergência só nas bandas finas das pontas (poucas amostras — olhe as barras de contagem) é só suporte ali.
